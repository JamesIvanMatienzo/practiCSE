Project Scope: Build a production-ready Android app for Civil Service Exam preparation with a focus on high readability, offline-first performance, and AI-assisted learning.
1. Data Architecture (Room SQL)
Entity Question:
id: Int (Primary Key).
track: String ("Professional" or "Sub-Professional").
category: String (Math, Logic, Constitution, etc.).
passage: String? (Optional header for reading comprehension).
questionText: String.
options: List<String> (Stored via TypeConverter).
correctIndex: Int.
aiLogic: String? (Cached response from Gemini).
Entity UserSession:
Stores lastTrack, lastCategory, and lastQuestionIndex to power the "Resume" button.
2. UI State & Momentum Logic
ViewModel Requirements:
Expose a StateFlow containing the current list of questions for the session.
Implement an onAnswerSelected(index: Int) function. This must:
Save the result to a UserProgress table.
Wait 300ms (for visual feedback).
Emit a side-effect to the UI to trigger pagerState.animateScrollToPage().
Jetpack Compose Components:
Screen: ExamScreen: Use a Scaffold. Place the passage (if present) in a TopAppBar or a pinned Box with verticalScroll. Place questions in a HorizontalPager.
Screen: Dashboard: Use LinearProgressIndicator for each subject performance.
3. The AI Integration (Gemini)
Prompt Template: Use a system prompt: "You are a Civil Service Exam tutor. Explain the logic of this question step-by-step for a student. Keep it concise.".
Gatekeeping: Check connectivity using ConnectivityManager. If offline, disable the "Generate AI Logic" button and show a tooltip.
Caching Strategy: When aiLogic is fetched, perform a Room Update on that Question ID so it never hits the API again for that specific question.
4. Hybrid-Lite Sync Strategy
Seeding: On first launch, check if Room is empty. If so, parse assets/questions.json and insertAll.
WorkManager: Create a SyncWorker that runs with NetworkType.CONNECTED. It should:
Post all "Reported Errors" (where isFlagged == true) to the backend.
Post the user's latest scores to the Ranking table in your cloud DB.
5. Visual Design Tokens
Surface: #F8F9FA.
Buttons: #1A73E8 with rounded corners (Material 3).
Typography: Use MaterialTheme.typography.bodyLarge for questions to ensure readability.
