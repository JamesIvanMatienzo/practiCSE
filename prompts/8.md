Firebase Integration & SDK Wiring
"Act as a Senior Android Developer. We need to replace the reflection fallbacks with real Firebase SDK implementations for practiCSE.  

Key Requirements:

Dependency Wiring:

Add the Google Services classpath to the root build.gradle.kts.  

Apply the Google Services plugin and add the implementation for firebase-config-ktx (Remote Config) and firebase-crashlytics-ktx (Crashlytics) in the app-level build.gradle.kts.  

Application Startup:

In the PractiCSEApp.kt class (the Application class), initialize Firebase.  

Wire the RemoteConfigManager.fetchAndActivateRemote() call into the onCreate() method so configurations are fetched as soon as the app starts.  

Local Defaults:

Set up a remote_config_defaults.xml file with a default passing_threshold of 80% to ensure the app functions even before the first remote fetch completes.  

Verification:

Ensure the CrashReporter is no longer a mock and is correctly sending non-fatal logs to the Firebase dashboard when a database error or network failure occurs.  

Output: Provide the updated build.gradle.kts files (root and app) and the updated PractiCSEApp.kt initialization logic."
